# CPSIM Web and Office Automation

A beginner-friendly collection of web practice projects and Microsoft Office automation files built while learning HTML, CSS, JavaScript, Excel, Word, and PowerPoint workflows.

## Live Links

- GitHub Repository: https://github.com/fazal305/cpsim-web-and-office-automation
- Live Demo: Not available yet. The web projects can be opened locally in a browser.

## Overview

This repository combines two learning tracks: browser-based frontend practice and office productivity automation. It includes small HTML/CSS/JavaScript exercises, interactive mini projects, Excel sheets, Word reports, and a PowerPoint presentation.

The project is best understood as a learning archive that shows steady practice across web development and digital office tools.

## Features

- HTML structure and semantic element examples
- CSS layout practice with flexbox, grid, typography, spacing, and styling
- JavaScript mini projects such as billing, quiz, survey, clock, dice, coin, and to-do examples
- Image gallery and restaurant menu practice pages
- Excel automation sheets for attendance, fees, grades, salary, and customer-style records
- Word documents and academic-style reports
- PowerPoint presentation practice

## Tech Stack

- JavaScript
- Microsoft Excel
- Microsoft Word
- Microsoft PowerPoint

Getting Started
Clone or download the repository.
Open any .html file in a modern web browser.
For Office files, open the .xlsx, .docx, or .pptx files in Microsoft Office or a compatible editor.
Use VS Code if you want to inspect or edit the HTML, CSS, and JavaScript files.
Architecture Notes
This is a collection repo, so each file or small folder is mostly independent. The web-projects folder contains frontend practice pages and mini projects, while office-automation contains spreadsheet, document, and presentation examples.
The project does not require a build step, package manager, or backend server.
Accessibility
Several pages are beginner practice files, so accessibility is an area for improvement. Future improvements should focus on:
Clear page titles
Semantic HTML structure
Form labels
Useful image alt text
Keyboard-friendly controls
Better color contrast
Performance
The web files are static and lightweight. Performance can be improved further by:
Compressing large images
Removing duplicate demo files
Keeping JavaScript small and page-specific
Using consistent file naming
